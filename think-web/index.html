Web 学习心得

2017-08-27

最近在学习 Web 开发的知识, 把一些想法记录下来.

什么是 Web 呢? Web == World Wide Web(WWW) 有个中文名字叫 ‘万维网’.

还有一个网叫 ‘互联网’, 台湾叫 ‘網際網絡’, 英文原名叫 ‘Internet’, 所以另一个中文名字叫 ‘因特网’. Web 和 Internet 是不一样的. 简单来说, Web 是建立在 Internet 上的一种服务, 建立其上的服务还有 FTP, SMTP, Telnet 等, 都是 Internet 的一部分, 只不过我们使用 Web 更多一些.

Web 是由英国科学家 Tim Berners-Lee 在 1989 年发明的, 这个发明是什么意思呢? 其实就是做了一点小小的贡献, 主要干了三件事:

  1. 定义了 HTTP 协议, 也就是规定了浏览器和服务器如何通信
  2. 写出了第一个 Web browser (网页浏览器)
  3. 写出了第一个 Web server (Web 服务器)

这就是 Web, 两个软件加上它们之间的交流语言.

我们常听到说前端工程师, 后端工程师. 前端主要与浏览器打交道, 为了让用户得到更好更快的网页, 由于与用户接近所以称之为前端. 后端主要与服务器打交道, 主要的工作内容是处理业务逻辑, 管理数据和提高性能和稳定性等.

近几年还出现了 ‘全栈工程师’ 这么个称号, 这个称号定义不明, 好像会写个网站的前端后端或者再会写个APP就是全栈工程师了, 在我看来是特别忽悠的, 只是一个花俏的名字而已, 然后擅长忽悠的投机分子李笑来就拿着这个去骗钱了.

我更喜欢的头衔是 ‘Web Developer’, Tim Berners-Lee 在某次访谈中的介绍很低调的写了这个, 不过人家是发明 Web 的开发者, 哈哈确实没毛病.

嗯, 回到 Web 本身, 学习新技术首先应该了解它解决了什么问题, 理解它的本质. 这也是本文的重点, 当然目前我只是一个初学者, 所以并没有什么深刻的看法.

我认为 Web 的本质就是 传输文件, 什么文件呢? HTML 文本和一些图片视频等媒体文件. 在 Web 之前, 人们是怎么完成同样的功能呢? FTP. 想象这么一个场景, 为了取得其他计算机上的资源, 我要利用 FTP 建立连接, 然后下载到我的计算机上, 然后通过相应的软件查看, 哪怕这个东西只是一段文字, 我也要下载一个 txt 文件然后打开. 是的, 太麻烦了, 我不需要那个文件, 我只想看一下然后关掉, 我不想下载, 我想直接就能看.

嗯, Berners-Lee 也是这么想的, 然后发明了 Web, 我们想看什么, 打开浏览器直接看就好了.

一开始只是文字, 后来逐渐的支持了图片, 声音, 视频, 网站越来越丰富. 这些内容通过 HTML 文件组织起来, 但人们不止要内容, 还要形式, 谁不爱美的东西呢, 所以就添加了一些样式在 HTML 里, 让网页更美观. 但样式越来越多, HTML 越来越臃肿, 对开发人员很不友好, 所以就把控制样式的代码剥离出来, 也就是 CSS. 但是这样的页面只是静态的, 对用户来说, 就是你看到的和我看到的一模一样, 对开发者来说, 更新页面就要再写个新的 HTML 文件. 要想有更多功能, 就要写动态的网站了, 也就是通过程序语言来控制输出什么内容, 比如我的微博和你的微博首页不是一样的, 是因为我们关注的人不是一样的, 后端把我们各自关注的人发的微博内容从数据库里拉出来, 然后发送给前端, 我们就看到了各自的首页. 后端处理了数据, 而前端控制了如何向我们展示这些数据. 后端常用的编程语言有 PHP, Java, Ruby, Python, Go 等.
